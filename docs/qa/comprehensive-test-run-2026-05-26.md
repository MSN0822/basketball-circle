## 発見した問題と対応

### 1. キャンセル時のイベントステータス再計算

事象:
- 定員2、閾値2のイベントでactiveを1名キャンセルすると、waitlistがactiveへ繰り上がって満員に戻る。
- しかし本番ではイベントステータスが `accepting` のまま残った。

影響:
- 実際には満員なのに受付中に見える可能性がある。

対応:
- `app/api/cancel/route.ts` を修正。
- 参加番号の正規化と繰り上げ処理が終わった後に、active数を再集計してイベントステータスを再計算するようにした。
- 修正後、ローカルの `lint` と `build` はPASS。
- デプロイ後再実行でPASS。

### 2. 不正な会員登録API入力が500になる

事象:
- 存在しない `auth_user_id` で `/api/members` を呼ぶと、DBの外部キー違反がそのまま500になった。

影響:
- クライアントにサーバーエラーとして見える。
- 入力不正として扱うべきケース。

対応:
- `app/api/members/route.ts` を修正。
- 外部キー違反を `400 auth_user_id が正しくありません` として返すようにした。
- テスト期待値も「400/401/403で拒否され、500にならない」に更新。
- 修正後、ローカルの `lint` と `build` はPASS。
- デプロイ後再実行でPASS。

### 3. 管理画面のLintエラー

事象:
- React hook lintで、`useEffect` 内の同期setStateと関数宣言順に関するエラーが出ていた。

対応:
- `app/admin/page.tsx`
- `app/admin/create/page.tsx`
- `app/admin/events/[id]/edit/page.tsx`

上記を修正し、`npm run lint` はPASS。
