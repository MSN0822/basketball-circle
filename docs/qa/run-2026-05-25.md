## 重要な検出事項

前回の複数人参加テストで、本格始動前に修正した方がよい不具合を確認した。

キャンセル待ちから繰り上がる処理自体は動作するが、キャンセル後に参加番号が詰め直されず、active参加者の `slot_number` が重複する。
原因候補は `app/api/cancel/route.ts` の `newSlot = current + 1` 周辺。

例:

- 初期状態: `1. A`, `2. B`, `3. C`, `待1. D`, `待2. E`
- Bキャンセル後: `1. A`, `3. C`, `3. D`, `待2. E`
- Aキャンセル後: `3. C`, `3. D`, `3. E`
